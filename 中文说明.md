# FigDataX — 高精度科学图表数据提取

**FigDataX** = **Fig**ure **Data** e**X**traction — 一个 agentic **Claude Code 技能**。

把从论文里裁剪/截图的图表图片（单张或多张）交给 Claude，它自动提取其中的数值，精度可达
**±0.5%**，并用验证叠加图自证。分工：引擎测**几何**（绘图区、刻度线像素位置、系列颜色、
亚像素质心），Claude 用视觉读**语义**（刻度值、图例），只有真正模糊时才会问你。

支持：散点图、折线图、柱状图（分组/堆叠）、箱线图、饼图、热图、极坐标图、误差棒、
对数轴、多面板图。输出：每图一个 CSV + 可选多 sheet Excel（含方法、校准 RMSE、
验证轮数等溯源信息）。

## 安装

```bash
git clone https://github.com/Shaowen-Ye/FigDataX ~/.claude/skills/FigDataX
bash ~/.claude/skills/FigDataX/scripts/setup.sh      # 用 uv 创建 .venv 并安装依赖
~/.claude/skills/FigDataX/.venv/bin/python -m scripts.figdatax self-test
```

> 目录名是 **`FigDataX`**（Linux 等大小写敏感系统需完全一致）。

## 使用

在 Claude Code 中直接说：

```
> 提取 /path/to/figure.png 图片数据
> 从 ./results/fig3.png 中读取数据
> 这几张图都提取一下，汇总成一个 Excel
```

Claude 按 [SKILL.md](SKILL.md) 的自主循环执行：识别图表类型 → 引擎几何检测
（`figdatax geometry`：绘图区 + 刻度位置 + 系列颜色）→ 亲自查看注释图核验 →
视觉读刻度值并配对 → 校准（RMSE < 1% 硬门槛）→ 亚像素提取 → 亲自查看验证叠加图 →
最多迭代 3 轮 → 交付 CSV/Excel + 溯源报告。多张图（≥3）会并行分派子代理批量处理。

也可直接用命令行（详见 [references/cli.md](references/cli.md)）：

```bash
PY="$HOME/.claude/skills/FigDataX/.venv/bin/python"
$PY -m scripts.figdatax geometry figure.png --json geom.json --annotate geom.png
$PY -m scripts.figdatax extract figure.png \
    --calibration-points calib.json --color-target 0 255 255 --subpixel --validate
```

## 核心原则

- **标记中心才是数据点。** 标记（圆、菱形、方、三角）的几何中心是真实值；大标记读边缘会引入
  5–10% 误差。用 `subpixel=True` 精修到质心。
- **不要猜 HSV。** 用 `suggest_series` 或 `pick_color(img, x, y)` 取真实颜色——猜测的 HSV
  是"0 检出"的首因。
- **人只在两个关口出现**：①放大后刻度仍不可读；②图例与颜色对应关系模糊。其余全自动。
- **OpenCV 陷阱**：图像是 BGR 非 RGB；色相 H∈[0,179] 非 0–360；像素 `img[y,x]` 行优先且 y 向下。
