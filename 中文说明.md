# FigDataX — 高精度科学图表数据提取

**FigDataX** = **Fig**ure **Data** e**X**traction

从科学论文图表（柱状图、折线图、散点图、箱线图、热力图、饼图、极坐标图、堆叠图）中精确提取
数值数据，以多点轴校准半自动方法为核心，精度可达 ±0.5%。本仓库同时提供 **Claude Code 技能**
与 **桌面 App**，两者关系见 [PROJECT.md](PROJECT.md)。

## 安装（技能）

```bash
git clone https://github.com/Shaowen-Ye/FigDataX ~/.claude/skills/FigDataX
bash ~/.claude/skills/FigDataX/scripts/setup.sh      # 用 uv 创建 .venv 并安装依赖
~/.claude/skills/FigDataX/.venv/bin/python -m scripts.figdatax self-test
```

> 目录名是 **`FigDataX`**（Linux 等大小写敏感系统需完全一致）。

## 使用

在 Claude Code 中直接说：

```
> 提取 /path/to/figure.png 图片数据
> 从 ./results/fig3.png 中读取数据
```

Claude 会按 [SKILL.md](SKILL.md) 的流程：查看图像 → 检测绘图区 → 多点轴校准 →
提取标记中心（亚像素）→ 在**输入图片所在目录**保存数据表与验证叠加图。

也可直接用命令行（详见 [references/cli.md](references/cli.md)）：

```bash
PY="$HOME/.claude/skills/FigDataX/.venv/bin/python"
$PY -m scripts.figdatax extract figure.png \
    --calibration-points calib.json --color-target 0 255 255 --subpixel --validate
```

## 核心原则

- **标记中心才是数据点。** 标记（圆、菱形、方、三角）的几何中心是真实值；大标记读边缘会引入
  5–10% 误差。用 `subpixel=True` 精修到质心。
- **不要猜 HSV。** 用 `pick_color(img, x, y)` 在真实标记上取色——猜测的 HSV 是"0 检出"的首因。
- **OpenCV 陷阱**：图像是 BGR 非 RGB；色相 H∈[0,179] 非 0–360；像素 `img[y,x]` 行优先且 y 向下。
